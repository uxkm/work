HTML

body와 인접한 자식 element는 들여쓰기에서 제외한다.

script, style, src에 쓰이는 모든 url은 https 프로토콜 기준으로 작성한다.

고정으로 사용되는 이미지 요소에는 alt attrbute를 반드시 작성한다.

inline style, inine style sheet, inline script는 demo code를 제외한 어떠한 경우에도 허용하지 않는다. (*모두 css, js 파일로 분리)

demo code는 별도의 css/js로 분리하지 않고 HTML 파일 내에 작성한다.

demo code는 블록화 하여 영역을 분리하고 반드시 주석을 작성한다.

CSS

기본 작성 규칙

css는 어떠한 경우에도 들여쓰기를 허용하지 않으며, 모든 속성값은 줄바꿈을 허용하지 않는다.

여러 Class에 동일한 속성을 부여할 경우, Class마다 줄바꿈을 허용한다.


.page1 .parent1 .class1,
.page1 .parent2 .class1{/* css style */}

색상은 HEXCODE로 작성하되, 투명도가 필요한 경우에만 rgba() 형태로 사용한다.

동일한 하위 Class명을 사용하더라도 css 중복 적용을 피할 수 있도록 body내 최상위 Element에 기준 Class를 작성한다.


1. order.html
~~
<body>
<div class="order">
    <div class="same-name-class">
 ~~//

2. order-complete.html
~~
<body>
<div class="order-complete">
    <div class="same-name-class">
 ~~//

order.css
.order .same-name-class{/* style */}
.order-complate .same-name-class{/* another style */}
속성 작성 규칙

작성 순서는 Element 속성 > Element 위치 > box 속성 > 콘텐츠 속성 순으로 작성한다.

box 속성은 외부에서 내부, 방향은 시계방향을 기준으로 작성한다.


.style{position:absolute;display:block;top:10px;left:5px;margin:10px 5px 0 5px;border:1px;padding:5px;font-size:15px;text-align:center;}

스타일 속성의 길이를 줄이기 위해 속성값 축약 사용 가능하다.


.bg-color{background-color:#fff;}
.p15{padding:15px;}

* 단 background-image 관련 항목은 축약하지 않는다.

Element 사이의 Margin은 후행 Element기준으로 작성하되, 처음 Element의 경우 first-child Pseudo class를 이용하여 보정한다.


<!-- content간 margin이 10px인 경우 -->
<div class="wrap">
    <div class="content">...</div>
    <div class="content">...</div>
    <div class="content">...</div>
</div>

<style>
.wrap .content{margin-top:10px;}
.wrap .content:first-child{margin-top:0;}
</style>

Box-sizing은 border-box로 선언한다.

z-index는 최소단위 10단위로 작성하며, 최댓값 9999 이내에서 결정한다.

파일 분류 정의

공통 Element style, Component style은 common.css로 별도 관리한다.

JS

기본 작성 규칙

함수, 변수명은 직관적으로 작성한다. 함수명은 가능하다면 반의어를 활용하여 작성한다.


function getElementWidth(){}
function setElementWidth(){}
function addListData(){}
function removeListData(){}

함수명 내에 hyphen 및 underscore, 숫자 사용을 금지한다.

함수, 변수명 명명 규칙은 카멜 표기법으로 통일한다.

jQuery object가 부여 된 변수는 $(변수명) 형태로 작성한다.

Initialize 함수 / page on load 함수는 항상 파일 최상단에 위치하도록 배치한다.

파일 분류 정의

공통 Component 관련 기능은 common.js로 별도 관리한다.

utility 함수의 경우 util.js로 별도 관리한다.

공통 라이브러리, 플러그인은 /{매체명}/js/co 하단에 위치하도록 한다.
